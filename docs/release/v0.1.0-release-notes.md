# VisDoc-Retrieve v0.1.0 Release Notes

Status: draft release notes. The `v0.1.0` tag has not been created and no
GitHub Release has been published.

## Summary

`v0.1.0` packages VisDoc-Retrieve as a completed evidence-driven retrieval
evaluation project. It includes the synthetic corpus, deterministic retrieval
baselines, mock visual MaxSim scaffold, hard-negative mining, training-readiness
gates, A100/tiny-runner evidence, dev-only pilot evaluation harness, frozen
final-comparison protocol, one-time final comparison, no-retune pledge, and
final presentation materials.

## Final Benchmark Boundary

Phase 6D completed a one-time frozen final comparison under
`phase-6b-final-comparison-protocol/v1`. The final test split was read once for
that authorized run. After that run, the no-retune pledge forbids tuning,
retrieval-pipeline changes, metric-definition changes, candidate-universe
changes, final-label changes, or result-driven reruns.

The final claim checklist records:

- `benchmark_claim_allowed=false`
- `claim_status=no_clear_improvement_claim`

This release does not claim benchmark improvement, model superiority, production
training readiness, or real visual-model performance.

## Systems

Systems run in Phase 6D:

- `bm25`
- `lexical_cosine`
- `bm25_lexical_rrf`
- `mock_visual`

Systems reported as unavailable:

- `tiny_lora_adapter`: `not_available`; no adapter checkpoint was committed.
- `zero_shot_visual_backend`: `not_available`; no real visual backend metrics
  were fabricated.

`mock_visual` is a deterministic CPU-only MaxSim scaffold. It is not a real
ColPali/ColQwen visual model result.
