# Final Comparison Protocol

This document freezes the Phase 6B protocol for any future final comparison.
It is protocol evidence only. It does not run training, read final-test files,
download models, use A100/GPU resources, create adapters, or claim benchmark
improvement.

## Protocol Status

- Protocol version: `phase-6b-final-comparison-protocol/v1`
- Change: `freeze-final-comparison-protocol`
- Status: protocol frozen, execution not started
- Final benchmark: not run
- Final test evaluation: not used
- Benchmark improvement claim: none
- Model weights committed: no
- Adapter checkpoints committed: no
- Training caches committed: no
- Private config or exact private model path committed: no

## Final-Test Access Gates

Final-test files MUST NOT be read by default validation, dev-only harnesses, or
training-pilot commands. A future final-test run requires a new OpenSpec change
that names:

- Frozen protocol version.
- Authorized final-test manifest and candidate universe.
- Exact command that will read final-test data.
- Model, adapter, or baseline source.
- Artifact hygiene scan.
- Public claim checklist.
- Reviewer approval record.

Without that later change, final-test status remains `not_run`.

## Comparison Table Schema

A final-comparison table MUST include these fields for every row:

- `method_id`
- `method_family`
- `split_scope`
- `candidate_universe`
- `artifact_source`
- `adapter_status`
- `status`
- `metrics`
- `final_test_used`
- `benchmark_claim`
- `evidence_path`
- `notes`

Rows that are unavailable MUST use `not_run`, `not_available`, or `blocked`
status. They MUST NOT carry fabricated metric values.

## Claim Checklist

A benchmark improvement claim is blocked unless all of these are satisfied:

- Final-test run authorized by a later OpenSpec change.
- Frozen protocol version referenced.
- Candidate universe fixed and recorded.
- Final-test metrics generated by the authorized command.
- Metric table includes baseline and model rows with matching split scope.
- No model weights, adapters, caches, private configs, or private paths are
  committed.
- Reviewer confirms no dev-only or tiny-runner evidence is being described as
  final benchmark evidence.
- README or public-facing docs include only claims supported by the frozen
  evidence.

## Next Allowed Steps

The next safe implementation step is either:

- controlled longer dev training under a new bounded change, still not final
  benchmark; or
- a final-comparison execution proposal that explicitly requests final-test
  access under this protocol.

Until then, public wording should remain evidence-readiness language, not
benchmark-improvement language.
