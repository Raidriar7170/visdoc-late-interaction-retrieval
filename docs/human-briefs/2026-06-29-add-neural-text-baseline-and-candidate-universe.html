Phase 2.5：Text Baseline 与 Candidate Universe 澄清

状态：archived，本地验证通过，OpenSpec canonical specs 已同步

一句话结论：本阶段只澄清文本基线命名和候选页集合：旧的 dense_text 已替换为 lexical_cosine，默认报告显式记录 evaluated_split_pages 候选集合，并加入只使用本地 deterministic stub 的 neural_text 诊断路径。

本阶段变化

  • 新增 candidate universe 配置：evaluated_split_pages、non_train_pages、all_pages。
  • 默认报告仍只评估 synthetic-smoke dev queries：24 个查询 rank 8 个 dev pages。
  • 文本方法 ID 现在是 bm25、lexical_cosine、neural_text、bm25_lexical_rrf、bm25_neural_rrf。
  • neural_text 使用 local_stub provider；外部 embedding、网络、GPU、model download、embedding cache 均为 false。
  • bm25_neural_rrf 只有在 neural_text 启用时才出现在报告中，避免把 lexical 和 neural hybrid 混在一起。

边界与风险

这是 Phase 2.5 文本诊断与候选集合澄清，不是真实 neural embedding 证据，也不是视觉模型结果。不要声称 ColPali/ColQwen、hard-negative mining、training、adapter、final test、benchmark improvement、retrieval improvement 或 model superiority。

Phase 3A 的 deterministic visual-smoke baseline 仍然存在；本阶段没有修改 visual retrieval/report code，也没有新增数据。
