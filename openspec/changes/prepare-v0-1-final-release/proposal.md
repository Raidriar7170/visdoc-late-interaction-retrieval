## Why

The repository has completed and archived the Phase 6D final comparison and
Phase 7A presentation package. Before creating a public `v0.1.0` tag or GitHub
release, the release evidence, checklist, and release-note wording need to be
prepared in a bounded reviewable PR.

## What Changes

- Add a `v0.1.0` release-notes draft with honest final-result boundaries.
- Add a tag/release checklist that keeps tag creation and GitHub Release
  publication behind explicit user confirmation.
- Add a machine-readable release-readiness report.
- Add a concise Chinese Human Brief for the release-prep phase.
- Update README/evidence/progress surfaces with release-prep links.
- Preserve the final comparison, no-retune, and no-overclaim boundaries.

## Capabilities

### New Capabilities

- `release-readiness`: Defines how final tag/release preparation is recorded
  without executing tag or publication actions.

### Modified Capabilities

None.

## Impact

Affected files are release documentation, release-readiness evidence,
OpenSpec change artifacts, README/evidence links, and progress-ledger metadata.
This change does not create a tag, publish a GitHub Release, deploy, train,
tune, rerun final comparison, read final-test labels/qrels, or modify final
metrics/rankings/manifest artifacts.
